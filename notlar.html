Big Idea 1: Collaboration and Program Development

Collaboration: Working together helps bring in different ideas and reduces mistakes. It's important to find good ways to work with others in coding so that you can solve problems better.

Computer Innovations: New inventions in computers can be both digital (software) or physical (hardware). We use many of them every day. To understand them, we should ask: Why were they made? What problems do they solve? What new things can they do?

Program Development: Developers think about what limits there are, what users need, and the goals. They plan and then build the program. Good documentation is important to track work and give credit to contributors.

Errors in Programming: Errors in coding include logic errors, syntax errors, runtime errors, and overflow errors. Testing and debugging are essential to fixing these mistakes.

Big Idea 2: Data Representation and Processing

Binary and Bits: Bits represent data in a computer system. Eight bits make a byte. Binary is a system that uses only two digits: "1" means the electrical signal is present, and "0" means the signal is absent.

Data Compression: Lossless data compression keeps the original data intact, while lossy compression greatly reduces file sizes at the expense of detail.

Data Challenges: Data sets often need to be merged, cleaned, and may contain errors or incomplete information that must be addressed.

Data Processing: Programs process data to help people gain insights. Programmers use tools to clean and filter data for analysis.

Big Idea 3: Algorithms and Programming

Assignments and Variables: A variable is a fundamental programming abstraction for storing and updating values. The assignment operator (‚Üê) assigns values to variables, enabling flexibility in managing program data.

Data Abstraction: Focuses on separating a data type's characteristics from its implementation. Examples include lists and strings for organizing and manipulating data.

Algorithms: A finite series of steps to solve a problem. It can be represented in pseudocode, diagrams, or natural language.

Strings: Strings are sequences of characters with operations like extracting substrings, searching for patterns, and concatenating or splitting text.

Boolean Expressions: Represent true/false values, essential for logical operations using operators like AND, OR, and NOT.

Conditionals and Iteration: Enables decision-making and repetition in programming through conditionals and loops.

Lists: Allow management of data by accessing, modifying, and determining the length of elements.

Binary Search: Halves sorted data iteratively to locate an element efficiently.

Procedures and Libraries: Reusable code blocks (procedures) and pre-written collections of code (libraries) simplify programming tasks.

Random Values and Simulations: Random values add variability, and simulations model real-world scenarios for analysis.

Algorithmic Efficiency: Algorithms are evaluated for execution efficiency: reasonable (polynomial) or unreasonable (exponential).

Big Idea 4: Computer Systems and Networks

Computing Devices: Computers and computing systems use hardware and software together to process and store data. Computing devices include both common devices (computers, tablets, phones) and embedded systems in everyday objects.

Hardware Components: Computer hardware consists of input devices (keyboard, mouse), output devices (monitor, speakers), processing units (CPU), storage (RAM, hard drives), and paths for transmitting data (buses).

Software: Programs that run on computing devices, including system software (operating systems) and application software (programs that help users perform tasks).

Internet: A global network of autonomous systems that allows computers to communicate. It's built on standard protocols that allow digital devices to send and receive data.

Network Protocols: Rules that define how data is transmitted across networks, including TCP/IP, HTTP, and DNS. These enable the internet's functionality and reliability.

Cybersecurity: The protection of computing systems against unauthorized access or attacks. Security measures include encryption, authentication, and authorization.

Fault Tolerance: The ability of a system to continue operating despite failures in some components, achieved through redundancy and error correction mechanisms.

Big Idea 5: Impact of Computing

Beneficial and Harmful Effects: Computing innovations can have both positive and negative impacts on society, economy, culture, and the environment. These effects must be carefully considered.

Digital Divide: Differences in access to computing devices and the internet, affecting economic, social, and educational opportunities for individuals and communities.

Computing Bias: Computing innovations can reflect and reinforce existing biases in society, affecting how they serve different populations.

Legal and Ethical Concerns: Computing raises issues related to privacy, intellectual property, security, and fair use. Laws and regulations govern many aspects of computing.

Personally Identifiable Information (PII): Information about individuals that can be used to track or identify them. Computing innovations often collect and use PII, raising privacy concerns.

Intellectual Property: Creative works protected by copyright, patent, and trademark laws. Digital media has complicated these protections through ease of copying and distribution.

Open Access: Movements advocating for free and open sharing of digital information, including open-source software, Creative Commons, and open data initiatives.

Crowdsourcing: Obtaining services, ideas, or content by soliciting contributions from a large group of people, especially online communities.
